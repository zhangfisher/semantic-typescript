**Semantic-TypeScript**  
*优雅、类型安全、异步流处理，专为现代 Web 而生*

---

### 概览

Semantic-TypeScript 是流处理技术的一次重大突破，它**融合**了 JavaScript `GeneratorFunction`、Java Streams 以及 MySQL 风格索引的最精华理念。其核心设计哲学简洁而强大：**通过精密的索引机制**，而非暴力迭代，构建极致高效的数据处理流水线。

传统库常常迫使你使用同步循环或笨拙的 Promise 链，而 Semantic-TypeScript 为当代前端开发量身打造了**完全异步**、函数式纯净且类型严谨的流处理体验。

在其优雅的异步模型中，数据**只有在上游显式调用 `accept`（以及可选的 `interrupt`）回调时**才会到达消费者。时机完全由你掌控——你需要它的时候，它就在那里。

---

### 为什么开发者如此喜爱它

- **零样板索引** —— 每个元素都天然携带索引或自定义索引  
- **纯函数式风格**，完整的 TypeScript 类型推断  
- **永不泄漏的事件流** —— `useWindow`、`useDocument`、`useHTMLElement`、`useWebSocket` 等  
- **开箱即用的统计能力** —— 支持 `number` 和 `bigint` 的平均值、中位数、众数、方差、偏度、峰度…  
- **可推理的性能** —— 根据需求自由选择有序或无序收集器

### 快速上手

```typescript
import { useOf, useRange, useWindow } from "semantic-typescript";

// 简单内存流
const numbers = useOf(10, 20, 30, 40)
  .map(n => n * 2)
  .toUnordered()                    // ← 终端操作前必须调用
  .summate();                       // 200n

// 带提前终止的无限范围
useRange(0n, 1_000_000n)
  .filter(n => n % 17n === 0n)
  .limit(10n)                       // 仅取前 10 个元素
  .toUnordered()
  .toArray();

// 实时窗口大小变化（5 个事件后自动停止）
useWindow("resize")
  .limit(5n)                        // ← 事件流必须显式终止！
  .toUnordered()
  .forEach((ev, idx) => console.log(`第 ${idx} 次调整窗口`));
```

### 核心概念

| 概念                                         | 用途                                                         | 适用场景                                             |
|----------------------------------------------|--------------------------------------------------------------|------------------------------------------------------|
| `AsynchronousSemantic`                       | 异步流、事件和惰性流水线的核心构建器                         | 实时事件、WebSocket、DOM 监听、长时或无限流          |
| `SynchronousSemantic`                        | 同步、内存内或循环式流的构建器                               | 静态数据、范围、立即执行的循环                       |
| `.toUnordered()`                             | 最快的终端收集器（基于 Map 索引）                            | 性能关键路径（O(n) 时间与空间，无排序）              |
| `.toOrdered()`                               | 带排序、索引稳定的收集器                                     | 需要稳定顺序或索引访问时                             |
| `.toNumericStatistics()` / `.toBigIntStatistics()` | 丰富的统计分析（均值、中位数、方差、偏度、峰度等）   | 数据分析与统计计算                                   |
| `.toWindow()`                                | 滑动窗口与翻滚窗口支持                                       | 时间序列处理、分批与窗口化操作                       |

---

**重要使用规则**

1. **事件流**（`useWindow`、`useDocument`、`useHTMLElement`、`useWebSocket` 等）返回 `AsynchronousSemantic`。  
   → 你**必须**调用 `.limit(n)`、`.sub(start, end)` 或 `.takeWhile()` 来停止监听，否则监听器将持续存在。

2. **终端操作**（`.toArray()`、`.count()`、`.average()`、`.reduce()`、`.findFirst()` 等）**仅在转换为收集器后可用**：
   ```ts
   .toUnordered()   // O(n) 时间与空间，无排序
   // 或
   .toOrdered()     // 已排序，约 O(n²) 时间
   ```

---

### 性能说明

| 收集器                          | 额外时间 | 额外空间 | 是否排序 | 最适合场景         |
|---------------------------------|----------|----------|----------|--------------------|
| `toUnordered()`                 | O(n)     | O(n)     | 否       | 追求极致速度       |
| `toOrdered()` / Statistics / Window | O(n²)    | O(n)     | 是       | 需要顺序与分析     |

需要极致速度时请选择 `toUnordered()`；仅当需要稳定排序或依赖有序数据的统计方法时，才切换到 `toOrdered()`。

**Semantic-TypeScript 与其他前端流处理器的对比**

| 特性                                 | Semantic-TypeScript                                                                 | RxJS                                                                 | 原生 Async Iterators / Generators            | Most.js                                                              |
|--------------------------------------|-------------------------------------------------------------------------------------|----------------------------------------------------------------------|---------------------------------------------|----------------------------------------------------------------------|
| **TypeScript 集成**                  | 一流原生支持，深度类型 + 内置索引感知 + 符号运行时保护                              | 优秀，但泛型复杂                                                     | 良好，需手动类型                            | 强，函数式优先                                                       |
| **内置统计分析**                     | 原生全面支持（均值、中位数、众数、方差、标准差、偏度、峰度、频率），支持 `number` 与 `bigint` | 原生不支持（需自定义操作符或 `reduce`）                              | 无                                          | 无                                                                   |
| **索引与位置感知**                   | 原生强大 bigint 索引（MySQL 风格）                                                  | 需自定义操作符（`scan`、`withLatestFrom`）                           | 需手动计数器                                | 基础，无内置索引                                                     |
| **事件流管理**                       | 专用类型安全工厂函数 + 显式提前终止控制                                             | `fromEvent` 强大，但需手动管理订阅                                   | 手动监听 + 取消                             | `fromEvent` 良好，轻量                                               |
| **性能与内存效率**                   | 卓越 —— 优化的 `toUnordered()`（O(n) 时间/空间，无排序）和 `toOrdered()` 收集器     | 很好，但操作符链增加开销                                             | 极佳（零开销）                              | 极佳                                                                 |
| **打包体积**                         | 极轻量                                                                              | 较大（即使树摇后）                                                   | 零（原生）                                  | 小                                                                   |
| **API 设计哲学**                     | 函数式收集器模式 + 显式索引与短路机制                                               | 响应式 Observable 模式                                               | 迭代器 / 生成器模式                         | 函数式，无点式                                                       |
| **提前终止与控制**                   | 优雅且显式（`interrupt`、`.limit()`、`.takeWhile()`、`.sub()`）                     | 良好（`take`、`takeUntil`、`first`）                                 | 手动（`for await…of` 中的 `break`）         | 良好（`take`、`until`）                                              |
| **同步与异步支持**                   | 统一 API —— 一流同步与异步流                                                       | 主要异步（同步支持有限）                                             | 两者均可，但需手动处理                      | 主要异步                                                             |
| **学习曲线**                         | 对喜爱函数式与索引流水线的开发者非常友好                                            | 较陡（大量操作符、热/冷 Observable、调度器）                         | 低                                          | 中等                                                                 |

**Semantic-TypeScript 的核心优势**

- 独有的内置统计与索引能力，无需手动 `reduce` 或额外库  
- 对事件流的显式控制，有效防止 RxJS 常见的内存泄漏  
- 同步与异步双模式，在单一一致的 API 中兼得两者之长  

这份对比清晰说明了：Semantic-TypeScript 特别适合现代 TypeScript 前端项目 —— 它在要求高性能、类型安全和丰富分析能力的同时，避免了传统响应式库的繁文缛节。
